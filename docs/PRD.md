# 默认值自定义 PRD

## 产品定位

默认值自定义是飞书多维表格字段捷径插件。第一版不做侧边栏，只为新建字段填充来自同表普通字段或查找字段的默认值。

## 开发指南校准

参考字段捷径公式版开发指南后，第一版正式提交物调整为公式版字段捷径 JSON，路径为：

```text
field-shortcut-formula/default-value-customization.json
```

公式版字段捷径通过公式字段实现默认值引用。它会随来源字段变化实时计算，不是一次性写入普通字段值。因此，本文后续关于“手动刷新”“自动新增记录填充”“选项自动创建”的内容只适用于普通前端插件或后续服务端/工作流方案，不属于公式版字段捷径第一版提交范围。

## 第一版范围

- 只支持新建字段捷径，不支持把已有字段转换成该类型。
- 公式版输出为公式字段，随来源字段实时计算。
- 字段配置只选择来源字段，不提供目标类型选择。
- 目标字段类型默认跟随来源语义；实际类型转换由用户通过飞书多维表格原生字段类型能力完成。
- 不提供手动刷新历史记录。
- 不提供一次性写入普通字段值。
- 不提供自动创建单选/多选选项。
- 不承诺后台批量写入自动化。

## 字段支持

字段捷径配置只提供“来源字段”选择。用户需要改类型时，通过飞书多维表格原生字段类型能力完成，插件不提供目标类型选择。

来源字段支持同表普通字段和同表查找字段。查找字段只可作为来源，不可作为目标。跨表取值由用户先创建飞书原生查找字段。

公式版配置通过 `formulaTemplate` 引用来源字段，不做类型转换。

## 调试方式

字段捷径公式版不需要启动本地服务。调试步骤：

1. 打开字段捷径开发助手。
2. 粘贴 `field-shortcut-formula/default-value-customization.json`。
3. 点击“添加字段”。
4. 在字段配置面板选择来源字段。
5. 查看生成字段的结果。

## 前端实验方案

以下能力不属于公式版第一版提交范围，仅保留在 `web/` 实验实现中，用于后续评估普通前端插件、服务端或工作流方案：

- 手动刷新历史记录。
- 新增记录时一次性写入普通字段值。
- 单选/多选缺选项时自动创建选项。
- 来源变化后不自动同步。
- 大表刷新、停止执行、失败重试。

## 风险控制

公式版第一版不执行批量写入，因此不涉及回滚、续跑、批量失败重试等问题。

如果后续启用前端实验方案，仍需保留以下限制：

- 大表刷新显示分级提醒。
- 超过 50,000 条记录必须二次确认。
- 执行支持停止后续批次，不支持回滚、撤销、续跑或日志导出。
