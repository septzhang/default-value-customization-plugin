# 默认值自定义 PRD

## 目标

做一个飞书多维表格字段捷径。用户新建字段时，选择一个来源字段，新字段显示这个来源字段的值。

第一版按字段捷径公式版开发指南实现，提交物是：

```text
field-shortcut-formula/default-value-customization.json
```

## 第一版范围

- 只新建字段捷径，不支持把已有字段转换成插件字段。
- 配置面板只选择来源字段。
- 不提供目标类型选择。用户要改类型，使用飞书多维表格自己的字段类型转换。
- 输出是公式字段，会跟随来源字段变化。
- 不做历史记录刷新。
- 不做普通字段值写入。
- 不自动创建单选或多选选项。
- 不承诺后台批量任务。

## 字段规则

来源字段支持：

- 文本
- 数字
- 日期
- 单选
- 多选
- 复选框
- 查找字段

查找字段可以作为来源，但插件不配置关联路径。跨表取值先用飞书原生查找字段完成。

公式版通过 `formulaTemplate` 引用来源字段，不做类型转换。因为开发指南里 `dataType` 只能写一个输出类型，当前配置先用文本类型承载结果。后续如果飞书支持动态输出类型，再调整这部分。

## 调试

字段捷径公式版不需要本地服务。

1. 打开字段捷径开发助手。
2. 粘贴 `field-shortcut-formula/default-value-customization.json`。
3. 点击“添加字段”。
4. 在字段配置面板选择来源字段。
5. 查看生成字段的结果。

## 暂不做

这些能力先不放进公式版：

- 手动刷新历史记录
- 新增记录时一次性写入普通字段值
- 单选/多选缺选项时自动创建选项
- 来源变化后不自动同步
- 大表刷新、停止执行、失败重试

这些能力更适合普通前端插件、服务端任务或飞书工作流。项目里的 `web/` 只是实验实现，不作为第一版提交物。

## 风险

公式版不会批量写数据，所以没有回滚、续跑、批量失败重试的问题。

真正需要注意的是输出类型。当前 JSON 里 `dataType` 只能固定写一个类型，和“目标类型跟随来源字段”不是完全等价。现阶段的处理方式是：插件负责引用来源字段，类型调整交给飞书多维表格。
