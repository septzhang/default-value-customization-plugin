# 默认值自定义

飞书多维表格字段捷径插件。当前可提交的是公式版配置：

```text
field-shortcut-formula/default-value-customization.json
```

仓库地址：

```text
https://github.com/septzhang/default-value-customization-plugin
```

## 现在怎么用

这个项目里有两部分：

- `field-shortcut-formula/`：字段捷径公式版配置。用于飞书的“字段捷径开发助手”，也是当前提交材料。
- `web/` 和 `shared/`：前端实验版。它验证过批量写入、刷新历史记录、创建选项等逻辑，但不作为当前字段捷径公式版提交物。

如果只是调试字段捷径，直接用 JSON 文件，不用启动本地服务。

## 字段捷径公式版

公式版字段捷径本质上是一个公式字段。它通过 `formulaTemplate` 引用来源字段，字段值会跟着来源字段变化。

配置里只有一个字段捷径：默认值自定义。

用户只需要选择来源字段。支持的来源字段包括：

- 文本
- 数字
- 日期
- 单选
- 多选
- 复选框
- 查找字段

开发指南里的 `presetFieldConfig.fieldProperty.dataType` 只能写一个输出类型，不能跟着用户选择的来源字段动态变化。所以这里先用文本类型承载结果。用户要改成数字、日期、选项等类型时，在飞书多维表格里改字段类型。

公式版不做这些事：

- 手动刷新历史记录
- 一次性写入普通字段值
- 自动创建单选或多选选项
- 固定住旧值，避免来源变化后同步

这些能力要走普通前端插件、服务端或者工作流。

## 调试字段捷径

1. 打开字段捷径开发助手。
2. 打开 `field-shortcut-formula/default-value-customization.json`。
3. 把 JSON 粘贴到开发助手里。
4. 点击“添加字段”。
5. 在字段配置面板选择来源字段。
6. 查看生成字段的结果。

通过调试助手创建的字段，只对同样使用调试助手的用户可见。要给其他人用，需要按字段捷径插件流程提交发布。

## 调试前端实验版

只有调试 `web/` 前端实验版时才需要启动本地服务：

```bash
npm install
npm run dev
```

默认地址：

```text
http://127.0.0.1:5173
```

这个页面需要在飞书多维表格插件环境里打开。直接用浏览器访问，只能看到页面，拿不到飞书 SDK 上下文。

## 构建

```bash
npm run build
```

构建产物在 `dist/`。项目已经配置了：

- `package.json` 中的 `"output": "dist"`
- Vite 的 `base: "./"`

这两项是为了静态产物提交时路径不出问题。

## 测试

```bash
npm test
```

测试覆盖了这些规则：

- 默认字段名生成
- 目标类型从来源字段推导
- 来源为空时不清空目标字段
- 仅填充空值时跳过已有值
- 覆盖模式下相同值不重复写入
- 缺少单选/多选选项时生成创建计划
- 自动模式下缺选项时跳过
- 查找多值只写入多选目标
