# 默认值自定义

飞书多维表格字段捷径插件项目。项目包含两套实现材料：

- `field-shortcut-formula/`：按字段捷径公式版开发指南生成的 JSON 配置，可用于“字段捷径开发助手”调试和提交。
- `web/` + `shared/`：按原始 PRD 生成的纯前端实验实现，用于验证“手动刷新、批量写入、选项创建”等能力，不提供预览页面。

当前参考开发指南后，正式提交优先使用 `field-shortcut-formula/default-value-customization.json`。

GitHub 仓库：

```text
https://github.com/septzhang/default-value-customization-plugin
```

## 字段捷径公式版

公式版字段捷径通过 `fieldType: 20` 和 `formulaTemplate` 生成公式字段。它的行为是引用来源字段并实时计算，不是一次性写入普通字段值。

配置文件：

```text
field-shortcut-formula/default-value-customization.json
```

包含 1 个字段捷径：默认值自定义。

字段配置只选择来源字段，支持文本、数字、日期、单选、多选、复选框和查找字段。开发指南中的 `presetFieldConfig.fieldProperty.dataType` 只能声明一个输出类型，无法根据来源字段动态变化；当前配置先以文本类型承载公式结果。用户需要改变字段类型时，通过飞书多维表格原生字段类型能力完成转换。

公式版不支持：

- 手动刷新历史记录。
- 新增记录时一次性写入普通字段值。
- 单选/多选缺选项时自动创建选项。
- 来源变化后不自动同步。

这些能力需要普通前端插件、服务端或工作流实现。

## 字段捷径调试

字段捷径公式版不需要启动本地服务。调试时使用飞书文档里的字段捷径调试模板或“字段捷径开发助手”：

1. 打开字段捷径开发助手。
2. 打开本项目文件 `field-shortcut-formula/default-value-customization.json`。
3. 将 JSON 内容粘贴到开发助手编辑区。
4. 点击“添加字段”。
5. 在字段配置面板选择来源字段。
6. 查看生成字段的结果。

通过调试插件创建的字段，仅限同样使用调试插件的用户可见。给其他用户使用时，需要按字段捷径插件提交流程发布。

## 功能范围

以下范围对应 `web/` 实验实现，不是公式版字段捷径提交物。

- 字段捷径插件，不做侧边栏。
- 只支持新建字段，不支持把已有字段转换成该类型。
- 目标字段类型默认与来源字段相同；用户需要改类型时，通过飞书多维表格原生字段类型能力完成。
- 来源字段支持同表普通字段和同表查找字段。
- 查找多值只允许写入多选目标。
- 手动刷新支持“仅填充空值”和“覆盖已有值”。
- 单选/多选缺少选项时，手动刷新会先确认新增选项数量再创建。
- 自动新增记录模式下不自动创建选项，只记录跳过原因。
- 来源为空时始终跳过，不清空目标字段。
- 来源值和目标值相同时不重复写入。

## 本地开发

本地服务只用于调试 `web/` 前端实验实现，不用于字段捷径公式版 JSON 调试。

```bash
npm install
npm run dev
```

前端默认地址：`http://127.0.0.1:5173`

在飞书多维表格插件调试入口配置本地地址后，在真实多维表格环境中测试。浏览器直接打开只能查看页面，无法获得 SDK 上下文。

## 构建

npm run build

## 平台能力边界

公式版字段捷径的字段配置、UI schema、模板语法以开发指南为准。

`presetFieldConfig.fieldProperty.dataType` 在开发指南中只能声明单一输出类型。当前公式版配置将结果先作为文本承载；如果用户需要数字、日期、选项等类型，由用户在飞书多维表格中通过字段类型能力完成转换。

前端实验实现中的配置保存优先尝试字段级配置能力。如果字段捷径运行时不提供字段级配置 API，项目会降级到浏览器本地存储，并在界面提示“当前配置仅保存在本机浏览器”。

前端实验实现中的自动新增记录填充依赖字段捷径运行时是否提供前端事件或执行入口。第一版不承诺后台自动化，只承诺插件运行期间的填充能力。稳定后台自动化应另做服务端或工作流方案。
