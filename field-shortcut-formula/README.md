# 字段捷径公式版配置

本目录是按字段捷径公式版开发指南生成的可调试 JSON 配置。

## 文件

- `default-value-customization.json`：字段捷径公式版配置。

## 设计取舍

开发指南中的公式版字段捷径通过 `fieldType: 20` 和 `formulaTemplate` 生成公式字段。它适合“引用来源字段并实时计算”的场景，不支持普通前端批量写入，因此以下 PRD 能力不在公式版配置中实现：

- 手动刷新历史记录。
- 新增记录时一次性写入普通字段值。
- 单选/多选缺选项时自动创建选项。
- 来源变化后不自动同步。

这些能力需要普通前端插件、服务端或工作流实现。

## 输出类型

第一版提供一个字段捷径：默认值自定义。

字段配置只让用户选择来源字段，支持文本、数字、日期、单选、多选、复选框和查找字段。

开发指南说明 `presetFieldConfig.fieldProperty.dataType` 只能声明一个输出类型，无法根据用户选择的来源字段动态声明输出类型。因此配置里先以文本类型承载公式结果。用户需要改变字段类型时，通过飞书多维表格原生字段类型能力完成转换。

## 调试方式

1. 打开开发指南里的字段捷径调试模板或“字段捷径开发助手”。
2. 粘贴 `default-value-customization.json`。
3. 点击“添加字段”。
4. 在字段配置面板选择来源字段。
5. 查看字段捷径效果。

通过调试插件创建的字段，仅限同样使用调试插件的用户可见。需要给其他用户使用时，按指南提交字段捷径插件。
